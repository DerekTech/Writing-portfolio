Laravel Basics

This module takes a look at how Laravel works on the backend. Let's look at the obvious example of requesting a website's page by entering its URL into a web browser. That browser sends a request to the server that houses the website requested. And if the server uses Laravel as its backend, what happens behind the scenes is shown in the image below.

The server first receives the request and passes it onto a route file which in turn looks for the website’s URL and decides what action to take (process data, send data or retrieve data), according to the type of request sent.

Once that is sorted, the data is compiled into a view, which is usually an HTML template. This view is sent back to the server and then to the web browser that made the request.

Route

Let us take a practical look at how the route file sends a view back to the web browser, depending on the route received. For this case, we will refer to the route file in our pizzahouse project.

In web.php, the root route is defined using:

<?php
use Illuminate\Support\Facades\Route;

Route::get('/', function () {
    return view('welcome');
});

The welcome view is stored in resources/views/welcome.blade.php.

This file contains Blade syntax – Laravel’s templating engine.

We’ll explore Blade later — for now, know that Laravel compiles it into HTML.

Creating a New Route

Add a second route:

Route::get('/pizzas', function () {
    return view('pizzas');
});

Make sure pizzas.blade.php exists in the views folder. You can copy the structure from welcome.blade.php.

Returning Other Data

Laravel can return strings:

Route::get('/pizza', function () {
    return 'pizza';
});

Or JSON:

Route::get('/pizza', function () {
    return ['name' => 'veg pizzas', 'base' => 'classic'];
});

This sends a JSON response with content-type: application/json.
