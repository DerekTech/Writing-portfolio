Blade Basics

In the last module, we discussed how we can send data from our route handler into the view and then output the variable in our view file (pizzas.blade.php) for Laravel to send to the browser.

Now, the syntax we used to output our variables in the view is a blade syntax, and in this module, we will be looking at more blade syntax.

Blade is a template engine that makes the use of data and logic inside of a view easier as opposed to regular PHP. And for this course, we will be using Blade for all our views because it makes things easier for us.

What Laravel does with Blade is that it compiles it into HTML behind the scenes and sends the resulting web page to the browser as requested by the user.

Asides outputting variables, what else does Blade have to offer? There are many things blade can do, let's look at the more common ones;

The IF statement

This statement starts with @ and then if, that is, @if and closes with @endif. You'll see the at (@) sign in almost every blade directive you'll come across. The if statement takes in conditions, for example:


    @if($price > 15)
    <p>pizza is expensive</p>
    @endif

What these few lines of code mean is; if the pizza price is above 15, it will display "pizza is expensive" on the browser.

If we check our preview page, we will not notice any changes, and that's because our price in the route handler is set to 10. If we make it 17 and save, we should see the text in the preview page because it is over 15.

If we want to add another condition, let's say, we also want to print something if the price is less than 5. Here is how we'll do it:


    @if($price > 15)
    <p>pizza is expensive</p>
    @elseif($price < 5)
    <p>this pizza is affordable</p>
    @endif

In summary, if the first condition is true, the second condition doesn't run. But if the first condition is not true, it checks to see if the second condition is true and prints the text if it's true. And if both conditions are not true, nothing happens. However, to get something, we add @else, for example:


    @if($price > 15)
    <p>pizza is expensive</p>
    @elseif($price < 5)
    <p>this pizza is affordable</p>
    @else
    <p>this pizza is in the normal price range</p>
    @endif

The UNLESS statement

This is the opposite of the if statement. Let's look at an example.


    @unless($base == 'cheesy crust')
    <p>you don't have a cheesy crust</p>
    @endunless

What this means is that, unless the base is a cheesy crust, print the text in the p tag. In other words, it will only print the text in the p tag if the condition is false.

The PHP directive

This allows us to use regular PHP in Blade, to do this, you write your PHP in this tag. It starts with @php and closes with @endphp.

Why would you need to output PHP in blade?

While there are different things blade can do for us, PHP is broader, giving you more control. If the need arises, you only have to open the @php and do your thing.

For example:


    @php
    $name = 'shaun';
    echo ($name);
    @endphp

If you save this and preview, you'll see the name Shaun on your pizza list page.
